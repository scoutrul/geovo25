## Адаптивность

### Брейкпоинты
- **Mobile**: 360px (base)
- **Tablet**: 1024px (md/lg)
- **Desktop**: 1720px (xl)

### Ключевые адаптации
- ✅ Padding контейнеров: `px-4 md:px-10 xl:px-16`
- ✅ Gaps между секциями: `gap-16 md:gap-24 xl:gap-32`
- ✅ Колонки: flexbox с адаптивными направлениями
- ✅ Типографика: адаптивные размеры заголовков
- ✅ Sticky Summary на desktop: `xl:sticky xl:top-24`

## Типы блоков контента

### Реализованные типы:
1. **text** - простой текстовый блок
2. **text-image** - текст + изображение
3. **subsection** - подзаголовок + текст + опциональное изображение
4. **testimonial** - отзыв клиента в карточке

## Использование

### Переход на страницу кейса:
```javascript
// С главной страницы через CaseCard
<CaseCard slug="ag1" ... />

// Программно
router.push('/cases/ag1')

// Прямая ссылка
/cases/ag1
```

### Добавление нового кейса:
1. Добавить данные кейса в data.ru.json и data.en.json с уникальным slug
2. Создать данные кейса в CaseDetailPage.vue (или вынести в отдельный файл)
3. Добавить изображения в `public/assets/cases/{slug}/`

## Известные особенности

1. **Изображения**: В данный момент используются placeholder пути. Необходимо добавить реальные изображения.
2. **Данные кейсов**: Пока данные хранятся внутри CaseDetailPage.vue. Для масштабирования рекомендуется вынести в отдельные файлы.
3. **Следующий кейс**: Логика определения следующего кейса захардкожена. Можно улучшить, сделав автоматическое определение на основе порядка в списке.
